Android: Sistema Operativo

Introduzione

Android è un sistema operativo open-source sviluppato inizialmente da Android Inc. e acquisito da Google nel 2005. È progettato principalmente per dispositivi mobili come smartphone e tablet. È basato sul kernel Linux e offre un'ampia gamma di funzionalità che lo rendono uno dei sistemi operativi più utilizzati al mondo.

Come è Fatto

Android ha un'architettura a strati che include:

  • Linux Kernel: versione base 2.6 modificata da Google, gestisce driver hardware (WiFi, USB, Bluetooth, batteria, fotocamera, touchscreen...).
  • Android Runtime (ART / DVM): esegue le app tramite Dalvik Virtual Machine (con compilatore Just-In-Time) o ART (con compilazione Ahead-of-Time). Le app sono in formato .apk, simile ai file .jar.
  • Librerie: includono le Core Libraries per funzioni Java e librerie native per la piattaforma.
  • Application Framework: fornisce API Java. Comprende:
    • Activity Manager: gestisce schermate (activity) e interazione utente.
    • Window Manager: gestisce le finestre delle app e la loro visibilità a schermo.
  • Applicazioni: interfaccia utente e app preinstallate come telefono, browser e contatti.

Perché Ogni Versione Ha il Nome di un Cibo?

Google ha adottato una convenzione divertente per nominare le versioni di Android, utilizzando nomi di dolci e dessert in ordine alfabetico. Questo approccio ha reso le versioni di Android memorabili e uniche.

  • Android 1.5: Cupcake
  • Android 1.6: Donut
  • Android 2.0-2.1: Eclair
  • Android 2.2: Froyo
  • Android 2.3: Gingerbread
  • Android 3.0: Honeycomb
  • Android 4.0: Ice Cream Sandwich
  • Android 4.1-4.3: Jelly Bean
  • Android 4.4: KitKat
  • Android 5.0: Lollipop
  • Android 6.0: Marshmallow
  • Android 7.0: Nougat
  • Android 8.0: Oreo
  • Android 9.0: Pie

Come è Realizzato

Android è progettato per essere modulare e scalabile. Gli sviluppatori possono creare applicazioni usando Java, Kotlin o C++. Il sistema sfrutta API dell'Application Framework e librerie runtime per permettere il funzionamento delle app su un'ampia varietà di hardware. Inoltre, ogni app gira in un processo separato grazie alla gestione della Dalvik/ART, garantendo sicurezza e isolamento.
